# فصل ۷ — اولین ورود و آشنایی با Dashboard

## هدف این فصل

در فصل قبل CCC را نصب کردیم.

اکنون برای اولین بار وارد Dashboard خواهیم شد.

در پایان این فصل:

✓ وارد CCC خواهید شد.

✓ ساختار Dashboard را خواهید شناخت.

✓ تفاوت بخش‌های Dashboard ،System و Settings را خواهید دانست.

✓ با Contribution Advisor آشنا خواهید شد.

✓ مفهوم Traffic ،Regions و Node Status را خواهید فهمید.

✓ خواهید دانست هر قابلیت در کدام بخش قرار دارد.

## 7.1 اولین ورود

**هدف**

ورود به Dashboard برای اولین بار.

**آدرس Dashboard**

اگر در فصل قبل از:

conduit.example.com

استفاده کرده باشید:

Dashboard از طریق:

https://conduit.example.com

در دسترس خواهد بود.

**صفحه Login**

پس از باز کردن آدرس فوق صفحه Login نمایش داده می‌شود.

**اطلاعات ورود**

از اطلاعاتی استفاده کنید که هنگام نصب تعریف کرده‌اید:

Admin Username

Admin Password

![صفحهٔ ورود CCC](../../screenshots/login-page.png)

*صفحهٔ ورود CCC.*

## 7.2 امنیت ورود

**هدف**

درک رفتار امنیتی سیستم.

**تعداد تلاش‌های ناموفق**

CCC از حملات حدس رمز عبور محافظت می‌کند.

**محدودیت**

پس از:

5

تلاش ناموفق:

حساب برای:

15 Minutes

قفل می‌شود.

**اگر قفل شدید چه کار کنید؟**

از طریق SSH به Raspberry Pi متصل شوید.

اجرای دستور:

sudo ccc-unlock admin

**نکته مهم**

به جای:

admin

باید نام واقعی کاربر مدیر را وارد کنید.

**اعتبارسنجی**

پس از Unlock باید بتوانید دوباره Login کنید.

## 7.5 Contribution Advisor

**هدف**

کمک به بهینه‌سازی میزان مشارکت ایستگاه.

Advisor به صورت خودکار وضعیت ایستگاه را تحلیل می‌کند.

**موارد بررسی شده**

نمونه:

CPU Usage

RAM Usage

Temperature

Client Capacity

Activity Patterns

**انواع پیشنهادها**

از کم‌اهمیت تا مهم:

Info

Suggestion

Strong Suggestion

Warning

**مثال**

اگر دمای Raspberry Pi زیاد شود:

Advisor ممکن است هشدار نمایش دهد.

اگر ظرفیت کلاینت‌ها نزدیک سقف باشد:

ممکن است پیشنهاد افزایش ظرفیت نمایش داده شود.

**نکته مهم**

Advisor فقط پیشنهاد می‌دهد.

هیچ تغییری به صورت خودکار اعمال نمی‌شود.

## 7.6 Node Status

**هدف**

نمایش وضعیت فعلی Conduit.

**وضعیت‌های ممکن**

Live

Starting

Disconnected

Offline

Unknown

**معنی وضعیت‌ها**

**Live**

Conduit در حال اجرا و پاسخگو است.

**Starting**

سرویس در حال راه‌اندازی است.

**Disconnected**

ارتباط با سرویس برقرار نیست.

**Offline**

سرویس فعال نیست.

**Unknown**

وضعیت قابل تشخیص نیست.

## 7.22 بخش Software Updates

**هدف**

بررسی و نصب به‌روزرسانی‌های Conduit Control Center از داخل Dashboard، بدون استفاده از خط فرمان. به‌روزرسانی همیشه توسط شما آغاز می‌شود — هیچ به‌روزرسانی خودکار یا پس‌زمینه‌ای وجود ندارد.

**محدوده**

این بخش فقط Conduit Control Center را به‌روزرسانی می‌کند. هستهٔ Psiphon Conduit (Core) را به‌روزرسانی نمی‌کند؛ آن جداگانه مدیریت می‌شود.

**چه چیزی نمایش داده می‌شود**

- Current — نسخه‌ای که اجرا می‌کنید.
- Latest — جدیدترین نسخهٔ پایدار موجود.
- Status — اینکه به‌روز هستید یا به‌روزرسانی در دسترس است.
- Last checked — آخرین باری که CCC با GitHub تماس گرفته است.
- What's new — پیش‌نمایش کوتاهی از یادداشت‌های انتشار.

به‌روزرسانی‌ها فقط از انتشارهای رسمی و پایدار منتشرشده می‌آیند.

**بررسی به‌روزرسانی‌ها**

CCC حداکثر روزی یک‌بار به‌صورت خودکار بررسی می‌کند. همچنین می‌توانید هر زمان Check Now را انتخاب کنید.

**نصب به‌روزرسانی**

وقتی به‌روزرسانی در دسترس است، Install Update را انتخاب کنید. CCC نسخهٔ جدید را دانلود و اعمال می‌کند و Dashboard به‌طور خودکار راه‌اندازی مجدد شده و دوباره متصل می‌شود — پس از مکثی کوتاه نسخهٔ جدید نمایش داده می‌شود. تنظیمات، گواهی TLS و داده‌های شما حفظ می‌شوند. اگر مشکلی پیش بیاید، CCC به‌طور خودکار به نسخهٔ قبلی بازمی‌گردد.

**یادداشت دربارهٔ Conduit Core**

اگر یک انتشار، نسخهٔ جدیدتری از Conduit Core را نسبت به نسخهٔ نصب‌شده توصیه کند، یادداشتی با گزینه‌های Continue Anyway و Cancel نمایش داده می‌شود. تصمیم با شماست. ادامه‌دادن فقط Conduit Control Center را به‌روزرسانی می‌کند — Conduit Core تغییر نمی‌کند.

**اگر GitHub در دسترس نباشد**

CCC آخرین نتیجهٔ شناخته‌شده را نمایش می‌دهد و به کار خود ادامه می‌دهد. همچنان می‌توانید به‌صورت دستی از طریق SSH به‌روزرسانی کنید.

**اولین به‌روزرسانی دستی است**

به‌روزرسان تک‌کلیکی تنها پس از اجرای نسخه‌ای که شامل آن است در دسترس قرار می‌گیرد. برای بار اول، یک‌بار از طریق SSH به‌روزرسانی کنید:

```bash
cd ~/conduit-control-center
git pull
sudo bash update.sh
```

پس از آن، به‌روزرسانی‌های بعدی را می‌توانید از همین صفحه انجام دهید.

### به‌روزرسانی نرم‌افزار

صفحهٔ **Software Updates** با یک اقدام CCC را از داشبورد به‌روزرسانی می‌کند و نشان می‌دهد که به‌روز هستید یا نه. CCC امضای هر به‌روزرسانی را پیش از اعمال تأیید می‌کند و اگر نسخهٔ جدید سالم نشود، به‌طور خودکار بازگردانی می‌کند. ببینید: **[به‌روزرسانی نرم‌افزار و انتشارهای امضاشده](software-updates-and-signed-releases.md)**.

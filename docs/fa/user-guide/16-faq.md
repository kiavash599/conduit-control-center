# فصل ۱۶ — پرسش‌های متداول (FAQ)

## هدف این فصل

این فصل پاسخ کوتاه و مستقیم به رایج‌ترین سؤالات اپراتورهای CCC را ارائه می‌دهد.

**نصب و سخت‌افزار**

**آیا Raspberry Pi 4 الزامی است؟**

خیر.

CCC روی Raspberry Pi 4 تست و توصیه شده است، اما محدود به آن نیست.

**آیا Raspberry Pi 3 قابل استفاده است؟**

بله.

اما توصیه می‌شود قبل از استفاده عملی، عملکرد آن را بررسی کنید.

**آیا 2 GB RAM کافی است؟**

بله.

CCC روی Raspberry Pi 4 با 2 GB RAM با موفقیت اجرا شده است.

**آیا SSD لازم است؟**

خیر.

اما SSD معمولاً:

- پایداری بیشتر
- سرعت بهتر
- عمر طولانی‌تر

ارائه می‌دهد.

**آیا Wi-Fi قابل استفاده است؟**

بله.

اما برای یک نود دائمی:

Ethernet

توصیه می‌شود.

**شبکه و Cloudflare**

**چرا به Cloudflare نیاز دارم؟**

Cloudflare سه وظیفه مهم انجام می‌دهد:

- DNS
- TLS Termination
- محافظت از IP واقعی سرور

**چرا Proxy باید روشن باشد؟**

در v0.3.0 نصب‌کننده بررسی می‌کند که:

Orange Cloud

فعال باشد.

**آیا IP استاتیک لازم است؟**

خیر.

DDNS برای همین طراحی شده است.

**آیا DDNS با IP داینامیک کار می‌کند؟**

بله.

اسکریپت DDNS هر 5 دقیقه IP عمومی را بررسی می‌کند.

**آیا DDNS از IPv6 پشتیبانی می‌کند؟**

خیر.

در v0.3.0 فقط:

A Record / IPv4

پشتیبانی می‌شود.

**Dashboard و عملیات**

**Dashboard باز نمی‌شود. چه کار کنم؟**

ابتدا:

sudo systemctl status conduit-cc

را بررسی کنید.

**تفاوت Health و Status چیست؟**

Health:

/api/health

فقط نشان می‌دهد CCC اجرا می‌شود.

Status:

/api/status

وضعیت واقعی نود و Broker را نشان می‌دهد.

**چرا Broker Disconnected نمایش داده می‌شود؟**

دلایل رایج:

- مشکل Conduit
- مشکل ارتباط با Broker
- توقف سرویس Conduit

**Contribution Advisor**

**آیا Advisor تنظیمات را خودکار اعمال می‌کند؟**

خیر.

Advisor فقط پیشنهاد می‌دهد.

**آیا Advisor کاربران را ردیابی می‌کند؟**

خیر.

CCC فقط از داده‌های Aggregate استفاده می‌کند.

**چرا Advisor هنوز پیشنهادی ارائه نمی‌کند؟**

ممکن است:

- تاریخچه کافی وجود نداشته باشد
- Traffic Collector فعال نباشد
- داده‌های 7 روزه هنوز جمع نشده باشند

**Personal Mode**

**آیا Personal Mode با ایجاد Identity فعال می‌شود؟**

خیر.

Identity به تنهایی کافی نیست.

باید:

Max Personal Clients > 0

تنظیم شود.

**آیا QR شخصی منقضی می‌شود؟**

خیر.

تا زمانی که Identity وجود داشته باشد معتبر است.

**آیا می‌توان Identity را حذف کرد؟**

خیر.

در v0.3.0 حذف مستقیم وجود ندارد.

**آیا Personal Mode در Backup ذخیره می‌شود؟**

تنظیمات ذخیره می‌شوند.

اما Identity واقعی ذخیره نمی‌شود.

**Ryve**

**Ryve چیست؟**

CCC فقط QR لازم برای Claim را تولید می‌کند.

Claim واقعی توسط برنامه Ryve انجام می‌شود.

**آیا QR Ryve حساس است؟**

بله.

باید مانند یک Credential خصوصی با آن رفتار شود.

**آیا Ryve در Backup ذخیره می‌شود؟**

خیر.

**آیا تولید QR باعث Restart می‌شود؟**

خیر.

**Backup & Restore**

**Backup شامل چه چیزهایی است؟**

به طور خلاصه:

- Database
- Configurations
- Settings

**Backup شامل چه چیزهایی نیست؟**

از جمله:

CF_API_TOKEN
SESSION_SECRET
TLS Private Key
Conduit Private Key
Ryve Identity

**اگر Passphrase را فراموش کنم چه می‌شود؟**

Backup غیرقابل بازیابی خواهد شد.

**آیا Restore فوراً انجام می‌شود؟**

خیر.

Restore در Background اجرا می‌شود.

**اگر Restore شکست بخورد چه می‌شود؟**

CCC تلاش می‌کند:

Automatic Rollback

انجام دهد.

**امنیت**

**چرا فقط یک Administrator وجود دارد؟**

نسخه v0.3.0 از مدل:

Single Administrator

استفاده می‌کند.

**آیا رمز عبور به صورت Plain Text ذخیره می‌شود؟**

خیر.

Hash ذخیره می‌شود.

**چرا CSRF استفاده شده است؟**

برای جلوگیری از درخواست‌های جعلی از سایت‌های دیگر.

**چرا Secretها در Backup ذخیره نمی‌شوند؟**

برای جلوگیری از انتقال یا افشای اطلاعات حساس.

**مدیریت پیشرفته**

**آیا می‌توان تعداد Workerها را افزایش داد؟**

خیر.

در v0.3.0:

workers = 1

باید حفظ شود.

**آیا Backup خودکار وجود دارد؟**

خیر.

Backup فقط به صورت دستی انجام می‌شود.

**آیا DDNS Log Rotation دارد؟**

خیر.

در v0.3.0:

/var/log/conduit-cc/ddns.log

به صورت خودکار Rotate نمی‌شود.

**آیا می‌توان فایل Drop-in را دستی ویرایش کرد؟**

ممکن است، اما توصیه نمی‌شود.

از Dashboard استفاده کنید.

**رمز عبور را فراموش کرده‌ام. چه کار کنم؟**

در صورت دسترسی SSH:

sudo ccc-unlock <username>

برای رفع Lockout استفاده می‌شود.

**بهترین توصیه برای اپراتورها چیست؟**

1. به‌روزرسانی‌های منظم انجام دهید.
2. قبل از Update، Backup بگیرید.
3. Passphrase را امن نگه دارید.
4. QRهای Personal و Ryve را محرمانه بدانید.
5. Dashboard را به صورت دوره‌ای بررسی کنید.
6. لاگ DDNS را مانیتور کنید.

# فصل ۹ — پیکربندی Conduit

## هدف این فصل

در فصل قبل با Contribution Advisor آشنا شدیم.

Advisor فقط پیشنهاد ارائه می‌دهد.

اما برای اعمال آن پیشنهادها باید از بخش:

Conduit Configuration

استفاده کنیم.

در پایان این فصل خواهید دانست:

✓ Configured و Effective چه تفاوتی دارند

✓ ظرفیت Conduit چگونه تنظیم می‌شود

✓ محدودیت پهنای باند چگونه تنظیم می‌شود

✓ Reduced Mode چیست

✓ تغییرات چگونه اعمال می‌شوند

✓ Rollback خودکار چگونه از سیستم محافظت می‌کند

## 9.1 Conduit Configuration چیست؟

**هدف**

آشنایی با صفحه تنظیمات Conduit.

این بخش برای مدیریت پارامترهای اصلی Conduit استفاده می‌شود.

در نسخه v0.3.0 تنظیمات زیر در این صفحه نمایش داده می‌شوند:

Max Common Clients

Bandwidth Limit

Reduced Mode

Max Personal Clients

**نکته مهم**

همه این موارد قابل ویرایش نیستند.

## 9.2 Configured و Effective

**هدف**

درک مهم‌ترین مفهوم این فصل.

در صفحه Configuration دو ستون مشاهده خواهید کرد:

Configured

Effective

**Configured چیست؟**

مقداری است که CCC ذخیره کرده است.

این مقدار از فایل‌های پیکربندی خوانده می‌شود.

**Effective چیست؟**

مقداری است که Conduit هم‌اکنون از آن استفاده می‌کند.

این مقدار از Metrics زنده Conduit دریافت می‌شود.

**مثال**

فرض کنید:

Configured

=

250

اما:

Effective

=

200

**آیا سیستم خراب است؟**

خیر.

ممکن است:

- Restart هنوز انجام نشده باشد
- Conduit هنوز با تنظیمات قبلی اجرا شود

این وضعیت طبیعی است.

## 9.3 Maximum Common Clients

**هدف**

تعیین حداکثر تعداد کلاینت‌های عمومی.

این پارامتر مشخص می‌کند چند کاربر عمومی می‌توانند همزمان از ایستگاه استفاده کنند.

**مثال**

50

به معنی:

Up To 50 Concurrent Public Clients

**نکته**

Advisor معمولاً روی همین مقدار پیشنهاد ارائه می‌دهد.

## 9.4 انتخاب مقدار مناسب

**هدف**

انتخاب ظرفیت منطقی.

**مقدار خیلی کم**

مثال:

10

ممکن است ظرفیت ایستگاه را محدود کند.

**مقدار خیلی زیاد**

مثال:

1000

ممکن است باعث فشار بیش از حد روی سخت‌افزار شود.

**توصیه**

در بیشتر موارد:

پیشنهادهای Advisor نقطه شروع خوبی هستند.

## 9.5 Bandwidth Limit

**هدف**

محدود کردن پهنای باند Conduit.

این مقدار مشخص می‌کند Conduit حداکثر چه میزان پهنای باند استفاده کند.

**مثال**

40 Mbps

به معنی:

Maximum 40 Megabits Per Second

**کاربرد**

زمانی که نمی‌خواهید Conduit تمام پهنای باند اینترنت را مصرف کند.

## 9.6 پهنای باند نامحدود

**هدف**

درک حالت Unlimited.

در CCC:

-1

به معنی Unlimited است.

**هشدار**

نامحدود بودن به معنی مصرف اجباری نیست.

Conduit فقط در صورت وجود تقاضا از پهنای باند استفاده می‌کند.

## 9.7 Reduced Mode چیست؟

**هدف**

کاهش محافظه‌کارانه ظرفیت در ساعات کم‌ترافیک.

Reduced Mode قابلیتی در Conduit است.

در این حالت:

- تعداد کلاینت‌ها کاهش می‌یابد
- محدودیت‌های محافظه‌کارانه‌تری اعمال می‌شوند

**هدف**

کاهش مصرف منابع در ساعات کم‌ترافیک.

## 9.8 ساعات Reduced Mode

**هدف**

درک نحوه تعریف بازه زمانی.

بازه Reduced Mode با:

Start Time

End Time

تعریف می‌شود.

**نکته بسیار مهم**

تمام زمان‌ها به صورت:

UTC

ذخیره می‌شوند.

**هشدار**

UTC با ساعت محلی شما متفاوت است.

در سوئد:

اختلاف زمانی بسته به فصل تغییر می‌کند.

بنابراین هنگام تنظیم ساعات باید به UTC توجه کنید.

## 9.9 Max Personal Clients

**هدف**

درک این پارامتر.

این مقدار در صفحه Configuration نمایش داده می‌شود.

اما:

Editable Here

=

No

**چرا؟**

زیرا مالک این تنظیم Personal Mode است.

برای تغییر آن باید به بخش Personal Mode مراجعه کنید.

## 9.10 اعمال تغییرات

**هدف**

درک فرآیند Apply.

وقتی روی:

Apply

کلیک می‌کنید:

CCC فقط فایل پیکربندی را تغییر نمی‌دهد.

بلکه فرآیند زیر اجرا می‌شود:

Validate

↓

Save

↓

Restart Conduit

↓

Health Check

↓

Success

## 9.11 Restart Conduit

**هدف**

درک دلیل Restart.

Conduit بسیاری از تنظیمات را فقط هنگام راه‌اندازی می‌خواند.

بنابراین:

Apply

↓

Restart

ضروری است.

**نکته**

اگر تغییری وجود نداشته باشد:

Restart انجام نمی‌شود.

## 9.12 Health Verification

**هدف**

اطمینان از سلامت سیستم پس از Apply.

بعد از Restart:

CCC منتظر می‌ماند تا Conduit دوباره سالم شود.

**معیار موفقیت**

موفقیت فقط به معنی اجرای Helper Script نیست.

موفقیت واقعی Conduit Healthy است.

## 9.13 Rollback خودکار

**هدف**

حفاظت از سیستم در برابر تنظیمات نامعتبر.

اگر Conduit پس از تغییرات سالم برنگردد:

CCC به صورت خودکار:

Rollback

انجام می‌دهد.

**فرآیند**

Apply New Settings

↓

Restart

↓

Health Check

↓

Failure

↓

Restore Previous Settings

↓

Restart

**مزیت**

کاهش احتمال از دست رفتن دسترسی به ایستگاه.

## 9.14 خطاهای اعتبارسنجی

**هدف**

شناخت خطاهای متداول.

**مقدار نامعتبر**

مثال:

Negative Client Count

رد می‌شود.

**زمان نامعتبر**

مثال:

25:99

رد می‌شود.

**فرمت نامعتبر**

هر مقدار خارج از محدوده مجاز پذیرفته نمی‌شود.

## 9.15 عیب‌یابی

**Configured و Effective متفاوت هستند**

بررسی کنید:

Restart Completed?

**تغییرات اعمال نمی‌شوند**

بررسی کنید:

Conduit Healthy?

**Reduced Mode رفتار مورد انتظار ندارد**

بررسی کنید:

UTC Times

را به درستی وارد کرده باشید.

**Rollback رخ داده است**

بررسی کنید:

System Logs

برای یافتن علت شکست.

## 9.16 نتیجه این فصل

اکنون می‌دانید:

✓ Configured چیست

✓ Effective چیست

✓ Capacity چگونه تنظیم می‌شود

✓ Bandwidth چگونه تنظیم می‌شود

✓ Reduced Mode چگونه کار می‌کند

✓ Apply Workflow چگونه اجرا می‌شود

✓ Rollback چگونه از سیستم محافظت می‌کند

**فصل بعد**

در فصل بعد:

Personal Mode

را بررسی خواهیم کرد و یاد خواهیم گرفت چگونه یک فضای خصوصی برای استفاده شخصی ایجاد کنیم.

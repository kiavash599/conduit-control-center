# فصل ۴ — مفاهیم پایه شبکه (بخش دوم)

**هدف این بخش**

در بخش اول یاد گرفتیم:

- IP چیست
- Public IP چیست
- Private IP چیست
- NAT چیست

اکنون می‌خواهیم یاد بگیریم:

- چگونه دستگاه‌ها IP دریافت می‌کنند
- چرا IP دستگاه‌ها ممکن است تغییر کند
- چرا Raspberry Pi باید همیشه IP ثابتی داشته باشد
- Port Forwarding چگونه کار می‌کند

## 4.5 DHCP چیست؟

**هدف**

درک نحوه اختصاص IP به دستگاه‌ها.

**DHCP به زبان ساده**

DHCP مخفف:

Dynamic Host Configuration Protocol

است.

این سرویس معمولاً روی روتر خانگی شما اجرا می‌شود.

**چه مشکلی را حل می‌کند؟**

فرض کنید:

- یک تلفن همراه جدید
- یک لپ‌تاپ جدید
- یک Raspberry Pi جدید

به شبکه متصل می‌شوند.

اگر DHCP وجود نداشت، باید به صورت دستی برای هر دستگاه IP تعیین می‌کردید.

DHCP این کار را به صورت خودکار انجام می‌دهد.

**مثال واقعی**

وقتی Raspberry Pi روشن می‌شود:

Raspberry Pi

│

▼

Request IP Address

│

▼

Router DHCP Server

│

▼

192.168.1.50

روتر یک IP در اختیار Raspberry Pi قرار می‌دهد.

**مزایا**

- مدیریت آسان‌تر شبکه
- عدم نیاز به تنظیم دستی IP
- کاهش خطاهای پیکربندی

## 4.6 چرا IP دستگاه‌ها تغییر می‌کند؟

**هدف**

درک رفتار طبیعی DHCP.

**نکته مهم**

DHCP معمولاً IPها را به صورت دائمی اختصاص نمی‌دهد.

بلکه برای مدت مشخصی اجاره می‌دهد.

این مدت:

Lease Time

نامیده می‌شود.

**مثال**

امروز:

Raspberry Pi

192.168.1.50

چند روز بعد:

Raspberry Pi

192.168.1.107

هر دو حالت ممکن است کاملاً طبیعی باشند.

**چرا این اتفاق می‌افتد؟**

زیرا DHCP وظیفه دارد IPهای آزاد را مدیریت کند.

اگر شبکه تغییر کند یا Lease منقضی شود، روتر ممکن است IP جدیدی اختصاص دهد.

**آیا این برای استفاده عادی مشکل است؟**

معمولاً خیر.

برای:

- لپ‌تاپ
- موبایل
- تبلت

تغییر IP اغلب اهمیتی ندارد.

اما برای Raspberry Pi که قرار است یک سرویس دائمی اجرا کند، موضوع متفاوت است.

## 4.7 DHCP Reservation چیست؟

**هدف**

ثابت نگه داشتن IP Raspberry Pi.

**مشکل**

در فصل قبل دیدیم که DHCP ممکن است IP دستگاه را تغییر دهد.

اما ما می‌خواهیم همیشه بدانیم Raspberry Pi روی چه IPی قرار دارد.

**راه‌حل**

DHCP Reservation

یا:

Static Lease

**DHCP Reservation چگونه کار می‌کند؟**

هر کارت شبکه یک شناسه منحصربه‌فرد دارد:

MAC Address

مثال:

B8:27:EB:12:34:56

روتر می‌تواند قانونی مشابه زیر داشته باشد:

اگر MAC Address برابر بود با:

B8:27:EB:12:34:56

همیشه IP زیر را اختصاص بده:

192.168.1.50

**نتیجه**

هر بار که Raspberry Pi روشن شود:

192.168.1.50

را دریافت خواهد کرد.

## 4.8 چرا Raspberry Pi باید همیشه IP ثابتی داشته باشد؟

**هدف**

درک مهم‌ترین دلیل استفاده از DHCP Reservation.

**مثال**

فرض کنید Port Forwarding را اینگونه تنظیم کرده‌اید:

Port 443

│

▼

192.168.1.50

همه چیز به درستی کار می‌کند.

اما چند روز بعد DHCP این IP را تغییر می‌دهد:

Raspberry Pi

192.168.1.107

در حالی که Port Forwarding هنوز به:

192.168.1.50

اشاره می‌کند.

**نتیجه**

اتصالات ورودی دیگر به Raspberry Pi نمی‌رسند.

ممکن است تصور کنید:

- Conduit خراب شده
- Cloudflare خراب شده
- CCC مشکل دارد

در حالی که مشکل فقط تغییر IP بوده است.

**قانون مهم**

⚠️

قبل از انجام Port Forwarding باید DHCP Reservation انجام شده باشد.

## 4.9 Port Forwarding چیست؟

**هدف**

اجازه دادن به روتر برای ارسال ترافیک ورودی به Raspberry Pi.

**مسئله**

در فصل اول NAT را یاد گرفتیم.

NAT باعث می‌شود دستگاه‌های داخل خانه از اینترنت قابل مشاهده نباشند.

این ویژگی برای امنیت بسیار مفید است.

**اما یک مشکل ایجاد می‌کند**

اگر شخصی از اینترنت بخواهد به Raspberry Pi متصل شود:

روتر نمی‌داند درخواست را به کدام دستگاه ارسال کند.

**Port Forwarding راه‌حل این مشکل است**

Port Forwarding به روتر می‌گوید:

اگر ترافیک روی این پورت دریافت شد

آن را به Raspberry Pi ارسال کن

**مثال**

Internet

│

▼

Router

Port 443

│

▼

192.168.1.50

**تشبیه ساده**

فرض کنید یک ساختمان بزرگ دارید.

تمام نامه‌ها ابتدا به نگهبانی ساختمان می‌رسند.

نگهبان باید بداند هر نامه را به کدام واحد ارسال کند.

Port Forwarding نیز نقش مشابهی دارد.

## 4.10 اعتبارسنجی اتصال

**هدف**

اطمینان از اینکه تنظیمات شبکه به درستی انجام شده‌اند.

**بررسی IP فعلی Raspberry Pi**

hostname -I

**بررسی MAC Address**

ip link

**بررسی دسترسی SSH**

از کامپیوتر:

ssh ubuntu@192.168.1.50

**مواردی که باید تأیید شوند**

✓ Raspberry Pi همیشه همان IP را دریافت می‌کند.

✓ SSH کار می‌کند.

✓ روتر DHCP Reservation را اعمال کرده است.

✓ Raspberry Pi در لیست DHCP Clients دیده می‌شود.

## 4.11 عیب‌یابی

Raspberry Pi هر بار IP متفاوتی می‌گیرد

بررسی کنید:

- DHCP Reservation ذخیره شده باشد.
- MAC Address صحیح انتخاب شده باشد.

**Port Forwarding کار نمی‌کند**

بررسی کنید:

- DHCP Reservation انجام شده باشد.
- IP صحیح Raspberry Pi انتخاب شده باشد.
- Rule روی روتر ذخیره شده باشد.

Raspberry Pi در DHCP Clients دیده نمی‌شود

بررسی کنید:

- کابل شبکه متصل باشد.
- Wi-Fi فعال باشد.
- دستگاه روشن باشد.

**نتیجه این فصل**

اکنون شما مفاهیم اصلی شبکه مورد نیاز برای راه‌اندازی CCC را می‌دانید:

✓ Public IP

✓ Private IP

✓ NAT

✓ DHCP

✓ DHCP Reservation

✓ Port Forwarding

این مفاهیم در فصل‌های بعدی بارها مورد استفاده قرار خواهند گرفت.

# فصل ۴ — مفاهیم پایه شبکه (بخش اول)

## هدف این فصل

در این فصل با مفاهیم پایه‌ای شبکه آشنا می‌شویم که برای راه‌اندازی موفق Conduit و CCC ضروری هستند.

**در پایان این فصل خواهید دانست:**

**✓ آدرس IP چیست**

✓ تفاوت Public IP و Private IP چیست

✓ چرا روتر خانگی شما NAT انجام می‌دهد

✓ چرا داشتن یا نداشتن Public IP برای Conduit اهمیت دارد

## 4.1 آدرس IP چیست؟

**هدف**

درک مفهوم آدرس IP و نقش آن در شبکه.

**چرا؟**

همان‌طور که هر خانه یک آدرس پستی دارد، هر دستگاه در شبکه نیز برای برقراری ارتباط به یک آدرس نیاز دارد.

**این آدرس در شبکه‌های کامپیوتری:**

**IP Address**

نامیده می‌شود.

**مثال واقعی**

**فرض کنید در خانه شما:**

- یک لپ‌تاپ
- یک تلفن همراه
- یک تلویزیون هوشمند
- یک Raspberry Pi

وجود دارد.

اگر هیچ‌کدام آدرس نداشته باشند، روتر نمی‌تواند تشخیص دهد که داده‌ها باید به کدام دستگاه ارسال شوند.

بنابراین هر دستگاه یک IP دریافت می‌کند.

**مثال:**

Laptop        192.168.1.10

Phone         192.168.1.20

TV            192.168.1.30

Raspberry Pi  192.168.1.50

**نکته مهم**

IP را می‌توان مشابه آدرس خانه در نظر گرفت.

اگر نامه‌ای برای خانه‌ای ارسال شود، آدرس مشخص می‌کند که نامه به کجا تحویل داده شود.

در شبکه نیز داده‌ها با استفاده از IP به مقصد صحیح ارسال می‌شوند.

## 4.2 Public IP چیست؟

**هدف**

درک آدرسی که اینترنت شما را در سطح جهانی شناسایی می‌کند.

**تعریف**

Public IP آدرسی است که ISP به اتصال اینترنت شما اختصاص می‌دهد.

این همان آدرسی است که سایر نقاط اینترنت شما را با آن می‌شناسند.

**مثال**

فرض کنیم ISP این آدرس را به شما اختصاص داده است:

185.201.45.123

**از دید اینترنت:**

You = 185.201.45.123

**مثال واقعی**

**وقتی وارد یک وب‌سایت می‌شوید:**

**https://example.com**

سرور مقصد معمولاً Public IP شما را مشاهده می‌کند.

نه IP لپ‌تاپ.

نه IP موبایل.

بلکه Public IP اتصال اینترنت شما.

**نکته مهم**

در اکثر خانه‌ها فقط یک Public IP وجود دارد.

در حالی که ممکن است ده‌ها دستگاه به شبکه متصل باشند.

## 4.3 Private IP چیست؟

**هدف**

درک آدرس‌هایی که فقط داخل شبکه خانگی استفاده می‌شوند.

**تعریف**

Private IP آدرسی است که داخل شبکه محلی شما استفاده می‌شود.

این آدرس‌ها در اینترنت قابل مسیریابی نیستند.

**محدوده‌های رایج**

بیشتر کاربران خانگی یکی از این محدوده‌ها را مشاهده می‌کنند:

192.168.x.x

**یا:**

**10.x.x.x**

**یا:**

172.16.x.x

**تا**

172.31.x.x

**مثال**

Laptop        192.168.1.10

Phone         192.168.1.20

Raspberry Pi  192.168.1.50

این آدرس‌ها فقط داخل خانه شما معنی دارند.

**نکته مهم**

ممکن است میلیون‌ها نفر در سراسر جهان از:

192.168.1.50

استفاده کنند.

این موضوع مشکلی ایجاد نمی‌کند زیرا این آدرس‌ها فقط در شبکه محلی معتبر هستند.

**تفاوت Public و Private IP**

| ویژگی | Public IP | Private IP |
| --- | --- | --- |
| در اینترنت قابل مشاهده است | بله | خیر |
| توسط ISP تخصیص می‌یابد | بله | خیر |
| توسط روتر تخصیص می‌یابد | خیر | بله |
| در سطح جهانی یکتا است | بله | خیر |

### 4.3.5 چرا Public IP برای Conduit مهم است؟

**هدف**

درک اهمیت Public IP در دسترسی به Raspberry Pi از اینترنت.

**چرا؟**

**در فصل‌های بعدی خواهیم دید که:**

- Port Forwarding
- DNS
- Cloudflare
- DDNS

همگی به نوعی به Public IP وابسته هستند.

**مثال**

فرض کنید شخصی در اینترنت می‌خواهد به گره Conduit شما متصل شود.

**اولین چیزی که باید پیدا کند:**

**Public IP شما**

است.

**اگر Public IP وجود نداشته باشد چه می‌شود؟**

**برخی ISPها از فناوری‌هایی مانند:**

**CGNAT**

استفاده می‌کنند.

**در این حالت ممکن است کاربر:**

- Public IP اختصاصی نداشته باشد.
- نتواند Port Forwarding انجام دهد.

**نکته مهم**

وجود Public IP یکی از مهم‌ترین پیش‌نیازهای راه‌اندازی موفق بسیاری از سرویس‌های اینترنتی است.

در فصل‌های بعدی روش تشخیص آن را بررسی خواهیم کرد.

## 4.4 NAT چیست؟

**هدف**

درک اینکه چگونه چندین دستگاه از یک Public IP مشترک استفاده می‌کنند.

**تعریف**

**NAT مخفف:**

**Network Address Translation**

است.

این فناوری معمولاً در روتر خانگی شما اجرا می‌شود.

**مسئله‌ای که NAT حل می‌کند**

**فرض کنید در خانه شما:**

**Laptop**

**Phone**

**TV**

**Raspberry Pi**

وجود دارد.

اما ISP فقط یک Public IP در اختیار شما قرار داده است.

**سؤال:**

**چگونه چهار دستگاه مختلف از یک Public IP استفاده می‌کنند؟**

**پاسخ:**

**NAT**

**NAT چگونه کار می‌کند؟**

وقتی Raspberry Pi به اینترنت متصل می‌شود:

192.168.1.50

روتر این درخواست را دریافت می‌کند.

سپس آن را با Public IP خانه شما به اینترنت ارسال می‌کند.

**بنابراین از دید اینترنت:**

185.201.45.123

در حال ارتباط است.

**نه:**

192.168.1.50

**تشبیه ساده**

فرض کنید یک شرکت فقط یک شماره تلفن اصلی دارد.

**اما داخل شرکت:**

- فروش
- حسابداری
- پشتیبانی

همگی از همان شماره اصلی استفاده می‌کنند.

NAT نیز عملکردی مشابه دارد.

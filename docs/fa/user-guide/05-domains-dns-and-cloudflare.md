## 5.1 Domain چیست؟

**هدف**

درک مفهوم دامنه و نقش آن در اینترنت.

**چرا؟**

فرض کنید بخواهید به وب‌سایتی مراجعه کنید.

معمولاً چیزی شبیه این وارد می‌کنید:

google.com

github.com

wikipedia.org

اما کامپیوترها در واقع با IP کار می‌کنند.

مثلاً:

142.250.74.14

به خاطر سپردن IPها دشوار است.

برای حل این مشکل از Domain استفاده می‌شود.

**Domain به زبان ساده**

Domain یک نام قابل فهم برای انسان است که به یک IP اشاره می‌کند.

مثال:

google.com

در واقع یک نام دوستانه برای یافتن سرورهای گوگل است.

**تشبیه ساده**

فرض کنید بخواهید با شخصی تماس بگیرید.

کدام راحت‌تر است؟

Ali

یا:

+46-72-xxx-xxxx

نام مخاطب برای انسان راحت‌تر است.

Domain نیز نقش مشابهی برای اینترنت دارد.

## 5.2 DNS چیست؟

**هدف**

درک سیستمی که Domain را به IP تبدیل می‌کند.

**DNS به زبان ساده**

DNS مخفف:

Domain Name System

است.

می‌توان DNS را دفترچه تلفن اینترنت در نظر گرفت.

**مثال**

وقتی در مرورگر می‌نویسید:

example.com

کامپیوتر شما از DNS می‌پرسد:

IP مربوط به example.com چیست؟

DNS پاسخ می‌دهد:

203.0.113.10

سپس مرورگر به آن IP متصل می‌شود.

**بدون DNS چه می‌شد؟**

باید IP تمام وب‌سایت‌ها را حفظ می‌کردید.

مثال:

185.199.108.153

140.82.121.3

104.18.32.7

که عملاً غیرممکن است.

## 5.3 Subdomain چیست؟

**هدف**

درک مفهوم زیر دامنه.

**مثال**

فرض کنید دامنه زیر را دارید:

example.com

می‌توانید زیر دامنه‌های مختلفی ایجاد کنید:

www.example.com

mail.example.com

vpn.example.com

conduit.example.com

**چرا Subdomain مهم است؟**

در این پروژه معمولاً از چیزی مشابه:

conduit.example.com

استفاده خواهیم کرد.

این نام به کاربران اجازه می‌دهد به جای حفظ IP از یک نام ثابت استفاده کنند.

**تشبیه ساده**

اگر Domain را یک ساختمان در نظر بگیریم:

example.com

Subdomainها واحدهای مختلف آن ساختمان هستند.

mail.example.com

support.example.com

conduit.example.com

## 5.4 Cloudflare چیست؟

**هدف**

آشنایی با سرویسی که در این راهنما استفاده خواهیم کرد.

**Cloudflare به زبان ساده**

Cloudflare مجموعه‌ای از خدمات اینترنتی ارائه می‌دهد.

از جمله:

- DNS
- امنیت
- TLS
- مدیریت دامنه
- API
- Dynamic DNS

**چرا CCC از Cloudflare استفاده می‌کند؟**

چند دلیل مهم:

**سادگی**

راه‌اندازی DNS بسیار آسان است.

**رایگان بودن**

بخش زیادی از امکانات مورد نیاز در پلن رایگان وجود دارد.

**API**

CCC می‌تواند به صورت خودکار رکوردهای DNS را به‌روزرسانی کند.

**پایداری**

Cloudflare یکی از بزرگ‌ترین ارائه‌دهندگان خدمات اینترنتی در جهان است.

**نکته مهم**

استفاده از Cloudflare توصیه می‌شود اما الزامی نیست.

در آینده ممکن است روش‌های دیگری نیز اضافه شوند.

## 5.5 Dynamic DNS چیست؟

**هدف**

درک مشکلی که Dynamic DNS حل می‌کند.

**مسئله**

در بسیاری از خانه‌ها ISP یک IP ثابت ارائه نمی‌دهد.

امروز:

185.10.20.30

فردا:

185.44.55.66

**مشکل**

اگر DNS همچنان به IP قدیمی اشاره کند:

کاربران دیگر نمی‌توانند به گره شما دسترسی داشته باشند.

**راه‌حل**

Dynamic DNS

**Dynamic DNS چگونه کار می‌کند؟**

CCC به صورت دوره‌ای Public IP را بررسی می‌کند.

اگر تغییر کرده باشد:

Cloudflare را به‌روزرسانی می‌کند.

در نتیجه:

conduit.example.com

همیشه به IP صحیح اشاره خواهد کرد.

## 5.6 API Token چیست؟

**هدف**

درک نحوه ارتباط CCC با Cloudflare.

**API چیست؟**

API راهی است که نرم‌افزارها از طریق آن با یکدیگر صحبت می‌کنند.

**مثال**

CCC می‌خواهد رکورد DNS را تغییر دهد.

برای این کار باید به Cloudflare درخواست ارسال کند.

Cloudflare باید مطمئن شود که این درخواست مجاز است.

**راه‌حل**

API Token

**API Token به زبان ساده**

API Token مشابه یک کلید دیجیتال است.

دارنده آن می‌تواند عملیات مشخصی انجام دهد.

**هشدار امنیتی**

⚠️

هرگز API Token خود را:

- در GitHub منتشر نکنید
- در تصاویر نمایش ندهید
- برای دیگران ارسال نکنید

## 5.7 حداقل دسترسی‌های مورد نیاز

**اصل مهم**

CCC از اصل:

Least Privilege

پیروی می‌کند.

یعنی فقط حداقل دسترسی مورد نیاز دریافت می‌شود.

**دسترسی‌های مورد نیاز**

معمولاً:

Zone

DNS

Edit

کافی است.

**چرا مهم است؟**

اگر Token به خطر بیفتد:

مهاجم فقط به DNS همان Zone دسترسی خواهد داشت.

نه به کل حساب Cloudflare.

## 5.11 رکورد DNS چیست؟

**مثال**

## 5.15 CCC چگونه از Cloudflare استفاده می‌کند؟

**هدف**

درک عملکرد واقعی DDNS.

**CCC چه اطلاعاتی نیاز دارد؟**

سه مقدار:

CF_API_TOKEN

CF_ZONE_NAME

CF_RECORD_NAME

**مثال**

CF_ZONE_NAME=example.com

CF_RECORD_NAME=conduit.example.com

**نکته مهم**

کاربر Zone ID را وارد نمی‌کند.

CCC آن را به صورت خودکار پیدا می‌کند.

**دلیل**

کاهش پیچیدگی پیکربندی.

## 5.16 Dynamic DNS چگونه کار می‌کند؟

**هدف**

درک فرآیند به‌روزرسانی خودکار.

**فرآیند**

هر ۵ دقیقه:

CCC:

1. Public IPv4 فعلی را بررسی می‌کند.
2. رکورد DNS فعلی را بررسی می‌کند.
3. آن‌ها را مقایسه می‌کند.

**اگر IP تغییر نکرده باشد**

no_change

ثبت می‌شود.

**اگر IP تغییر کرده باشد**

Cloudflare به‌روزرسانی می‌شود.

و:

updated

ثبت می‌شود.

## 5.17 اعتبارسنجی

**مشاهده تنظیمات**

sudo grep -E '^CF_(ZONE_NAME|RECORD_NAME)=' /etc/conduit-cc/.env

crontab -u conduit-cc -l

**مشاهده لاگ‌ها**

tail -n 20 /var/log/conduit-cc/ddns.log

sudo -u conduit-cc /usr/local/bin/cloudflare-ddns.sh

tail -n 1 /var/log/conduit-cc/ddns.log

## 5.18 نکته مهم درباره dig

بسیاری از کاربران انتظار دارند:

dig +short conduit.example.com

IP واقعی Raspberry Pi را نمایش دهد.

**اما این همیشه درست نیست**

اگر:

Proxy Status = Proxied

باشد:

Cloudflare ممکن است IPهای خودش را نمایش دهد.

**نتیجه**

این رفتار طبیعی است.

و نشانه خرابی DDNS نیست.

## 5.19 عیب‌یابی

**DDNS به‌روزرسانی نمی‌شود**

- Token معتبر باشد.
- رکورد DNS وجود داشته باشد.
- Cloudflare در دسترس باشد.

**نصب متوقف می‌شود**

بررسی کنید:

Proxy Status

=

Proxied

باشد.

**خطای Token**

بررسی کنید:

Zone → Zone → Read

Zone → DNS → Edit

اعطا شده باشند.

**لاگ خطا مشاهده می‌شود**

بررسی کنید:

tail -n 50 /var/log/conduit-cc/ddns.log

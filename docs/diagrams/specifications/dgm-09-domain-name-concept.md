## Purpose
Teach that a domain name is a human-friendly label that stands in for a numeric IP address.

## Audience
Beginner operator / non-network specialist.

## Required elements
- Nodes: `Domain name conduit.example.com`, `IP address 203.0.113.10`.
- Relationship: domain name —(easier to remember than a number)→ IP address.

## Visual direction
Two boxes, one relationship arrow. Minimal.

## Text labels
English, short.

## Security/privacy constraints
Generic example domain (`conduit.example.com`) and documentation IP (`203.0.113.10`, TEST-NET-3) only. No real domains, tokens, QR codes, usernames, or credentials.

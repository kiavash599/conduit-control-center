# Runbook — v0.3.18 release qualification (RPi2 + RPi4)

**Document type: qualification CONTRACT / checklist — not an executable ceremony script.**
It defines the required order, gates and pass criteria. It deliberately contains `<CANDIDATE>` and
path placeholders, because the exact Owner ceremony commands (SSH targets, absolute bundle paths,
resource limits, key paths) can only be generated once the candidate SHA exists and the Phase-A
reuse decision is made. Do not paste these steps as-is.

Owner-executed, in order. **Nothing here has been run.** Every step is gated: do not proceed past a
red gate. No step may be re-pointed at a different path "to make it work".

Conventions: `<CANDIDATE>` = the full 40-hex v0.3.18 candidate commit. Pi access is
`ssh -p 1222 kiavash@ssh.rockysystem.net`. Docker on the RPi2 is transient — verify stopped before,
stop and disable after, on every exit path.

## 1. Authoritative local validation, then push, then candidate CI

**The Pi must not be used to discover failures that local validation or candidate CI could have
found first.** Complete these before any hardware ceremony:

**Platform authority model.** The repository contains legacy POSIX/Linux tests (`/bin/bash`, POSIX
path semantics, `fcntl`, Linux service/helper/firewall/deployment shell behavior) that are invalid
on Windows. A full-Windows pytest run that fails on those is
`INVALID_AS_AUTHORITATIVE_PLATFORM_MISMATCH` — neither a PASS nor a product-regression FAIL.

1. Authoritative **Windows** run (PowerShell — one self-contained block; Ruff must pass before
   the focused pytest runs):

   ```powershell
   $Python = Join-Path (Get-Location) ".venv\Scripts\python.exe"
   if (-not (Test-Path -LiteralPath $Python -PathType Leaf)) {
       throw "ABORT: repository venv interpreter not found: $Python"
   }
   & $Python -m ruff check .
   $RuffExit = $LASTEXITCODE
   if ($RuffExit -ne 0) {
       throw "RUFF_FAIL: exit $RuffExit — fix lint before any test run"
   }
   $FocusedTests = @(
       "tests/unit/test_logical_tree.py",
       "tests/unit/test_transfer_manifest.py",
       "tests/unit/test_phase_b_to_release_e2e.py",
       "tests/unit/test_build_wheelhouse.py",
       "tests/unit/test_release_manifest.py",
       "tests/unit/test_update_verify.py",
       "tests/unit/test_builder_provenance.py",
       "tests/unit/test_ccc_verify_release.py",
       "tests/unit/test_release_canonical.py",
       "tests/unit/test_release_lock_drift.py",
       "tests/unit/test_release_preflight.py",
       "tests/unit/test_builder_scripts.py",
       "tests/unit/test_reuse_authz.py",
       "tests/unit/test_sanitize_target_tags.py",
       "tests/unit/test_acquire_reuse_wheels.py",
       "tests/unit/test_gen_active_inputs.py",
       "tests/unit/test_canonical_bytes.py",
       "tests/unit/test_update_taxonomy.py",
       "tests/unit/test_version.py",
       "tests/unit/test_api_update_pure.py"
   )
   & $Python -m pytest -q -p no:cacheprovider $FocusedTests
   $PytestExit = $LASTEXITCODE
   if ($PytestExit -eq 0) {
       "WINDOWS_FOCUSED=PASS (ruff=0, pytest=0, modules=$($FocusedTests.Count))"
   }
   if ($PytestExit -ne 0) {
       throw "WINDOWS_FOCUSED=FAIL: pytest exit $PytestExit"
   }
   ```

   The 20-module list deliberately EXCLUDES the four legacy modules the captured Windows log
   proved platform-incompatible — `test_arch_asset_mapping.py`, `test_ccc_apply_https_port.py`,
   `test_ccc_update_apply_audit.py`, `test_firewall_ssh_plan.py` (bash/helper gates without an
   `os.name`/`sys.platform` guard, so the WSL `bash.exe` launcher runs them against Windows
   paths) — and `test_ccc_update_apply_helper.py`, whose module-level
   `skipif(sys.platform != "linux")` yields zero executed Windows coverage.
   `test_builder_scripts.py` stays: only its native-bash cases skip; its platform-independent
   contract tests execute. All excluded modules run unchanged in the WSL/Linux full suite.
2. Authoritative **WSL/Linux** run: the FULL `pytest tests/unit/ tests/integration/` — including
   the POSIX/bash/fcntl and shell-behavior tests — plus the shell gate below. Linux coverage of
   the legacy tests is not weakened or skipped.

   **Shell gate.** `bash -n` on all three builder scripts, no exclusions; then ShellCheck with
   EXACT, pre-existing, justified baseline exclusions. These five findings predate v0.3.18
   (present at `aa6ae50`), are individually reviewed and intentional, and CI has never
   ShellChecked `release/builder/*.sh` — they are recorded debt, **not** permission for new
   warnings. Any OTHER finding, in any of the three files, fails the gate.

   - SC1091 (×2): both source sites carry `# shellcheck source=manifest-capture.lib.sh`; the
     library is dynamically located and is checked separately, directly, below.
   - SC2024 (×2, lib only): intentional — the ceremony user opens the mode-0600 output
     (`umask 077` subshell) while only the producing command runs under sudo, so the file is
     never root-created.
   - SC2015 (×1, lib only): the existing atomic-publish chain — `cp && mv || { cleanup; fail; }`
     is the intended "both must succeed, otherwise fail and clean up" semantics.

   The `-e` flag excludes a code for the WHOLE invocation, so on its own it would also hide a
   NEW finding with the same code. The gate below therefore first audits the RAW (no-exclusion)
   output against the exact five-line baseline — path, line and code — and only then runs the
   scoped zero-warning invocations. A new same-code finding changes the raw set and fails step 1.

   ```bash
   #!/usr/bin/env bash
   # v0.3.18 WSL shell gate -- exact-baseline audited, fail-closed. Run from the repo root.
   set -u
   command -v shellcheck >/dev/null || { echo "ABORT: shellcheck not installed"; exit 1; }
   E1=release/builder/build-builder-image.sh
   E2=release/builder/build-wheelhouse-offline.sh
   LIB=release/builder/manifest-capture.lib.sh

   # (0) bash -n on all three REAL scripts, no exclusions.
   bash -n "$E1" && bash -n "$E2" && bash -n "$LIB" || { echo "ABORT: bash -n failed"; exit 1; }

   # (1) EXACT raw-baseline audit (no exclusions, machine-readable GCC format).
   EXPECTED="$E1:29:SC1091
   $E2:102:SC1091
   $LIB:66:SC2024
   $LIB:76:SC2024
   $LIB:91:SC2015"
   raw="$(shellcheck --format=gcc "$E1" "$E2" "$LIB")"; rc=$?
   # 0 = no findings (baseline vanished: audit is stale), >1 = shellcheck error. Only 1 is valid.
   [[ $rc -eq 1 ]] || { echo "ABORT: raw shellcheck exit $rc, expected 1 (findings present)"; exit 1; }
   actual="$(printf '%s\n' "$raw" \
     | sed -nE 's/^([^:]+):([0-9]+):[0-9]+: +[a-z]+: .*\[(SC[0-9]+)\]$/\1:\2:\3/p' | sort)"
   nraw=$(printf '%s\n' "$raw" | grep -c .)
   nact=$(printf '%s\n' "$actual" | grep -c .)
   [[ "$nraw" -eq "$nact" ]] \
     || { echo "ABORT: $((nraw-nact)) unparseable shellcheck line(s):"; printf '%s\n' "$raw"; exit 1; }
   if ! diff <(printf '%s\n' "$EXPECTED" | sed 's/^ *//' | sort) <(printf '%s\n' "$actual"); then
     echo "ABORT: raw findings differ from the reviewed 5-item baseline (see diff above)"; exit 1
   fi

   # (2) Scoped zero-warning gates -- only reachable after the exact baseline held.
   shellcheck -e SC1091 "$E1" "$E2" || { echo "ABORT: entry-script gate failed"; exit 1; }
   shellcheck -e SC2024,SC2015 "$LIB" || { echo "ABORT: library gate failed"; exit 1; }

   # (3) Non-vacuity proof for BOTH exclusion configurations, on temp copies OUTSIDE the repo.
   tmpd="$(mktemp -d)"; trap 'rm -rf "$tmpd"' EXIT
   cp "$E1" "$tmpd/entry.sh";  printf 'rm -rf $UNQUOTED_VAR\n' >> "$tmpd/entry.sh"
   cp "$LIB" "$tmpd/lib.sh";   printf 'rm -rf $UNQUOTED_VAR\n' >> "$tmpd/lib.sh"
   if shellcheck -e SC1091 "$tmpd/entry.sh" >/dev/null 2>&1; then
     echo "ABORT: entry-script exclusion config is VACUOUS"; exit 1
   fi
   if shellcheck -e SC2024,SC2015 "$tmpd/lib.sh" >/dev/null 2>&1; then
     echo "ABORT: library exclusion config is VACUOUS"; exit 1
   fi
   echo "SHELL_GATE=PASS (baseline=5 exact, scoped gates clean, non-vacuity proven both configs)"
   ```

   Fail-closed properties: malformed/unparseable ShellCheck output fails the line-count check in
   step 1; a MISSING baseline finding fails the `diff` (or the exit-code check if all vanish); any
   EXTRA finding — including another occurrence of SC1091/SC2024/SC2015 — changes the raw set and
   fails the `diff` before any exclusion is applied; a vacuous exclusion configuration fails
   step 3. The temp directory is removed on every exit path via the EXIT trap.

## 2. Phase-A reuse-or-rerun gate

Phase A may be **reused** only if **all** of the following pass. Any failure ⇒ rerun Phase A.

1. All six image-context files byte-identical to the validated Phase-A context:
   `Containerfile`, `apt-packages.list`, `rustup-init.sha256`,
   `requirements-build-backends.lock`, `requirements-build-backends.source-allowlist`,
   `partition_backends.py`.
2. `builder-inputs.kv` passes the strict 16-key reader (`read_builder_inputs.py`).
3. Base-image digest continuity (`CCC_BASE_IMAGE_DIGEST` unchanged).
4. Tag → image-ID binding holds.
5. Manifest-digest no-drift on re-capture.
6. In-image `/opt/ccc` proof matches the six committed inputs (Phase B enforces this anyway).

The six-file check is a **pre-check**; the authoritative proof is the in-image binding Phase B
performs. Record the decision and its evidence in the continuation ledger.

## 3. Fresh Phase B from the exact candidate commit

Check out `<CANDIDATE>` on the Pi, confirm `git rev-parse HEAD` matches and the tree is clean, then
run the committed Phase-B entry point. Do **not** reuse the v0.3.17 bundle or provenance.

Phase B now preflights the **output pair** before the expensive build and refuses either collision:

    <out>/bundle                                    # must not exist
    <out>/phase-b-bundle-transfer-manifest.json     # must not exist

On success it publishes the bundle **and** generates + verifies the transfer manifest. If manifest
generation or verification fails, only this attempt's outputs are removed — pre-existing evidence is
never touched, and no bundle-only half-state remains.

Expected: 30 wheels (6 built + 24 reused), SHA256SUMS, runtime lock, format-3 provenance, build
evidence, `TRANSFER_MANIFEST=GENERATED` then `TRANSFER_MANIFEST=VERIFIED`.

Stop Docker, `docker.socket` and `containerd` afterwards and confirm `inactive/disabled`.

## 4. Transfer and mandatory Owner-side verification

Transfer the bundle **and** its manifest with `scp` (binary; no text-mode conversion). Then verify on
the Owner PC with the same committed contract:

    python -m release.transfer_manifest verify --bundle <bundle> --manifest <manifest>

Expect `TRANSFER_MANIFEST=VERIFIED`. This recomputes the Logical Tree Digest, enforces the exact
34-file set at every depth, and cross-checks SHA256SUMS, provenance, build evidence and the runtime
lock. Any failure stops the release.

## 5. Pi ↔ Windows Logical Tree Digest equality

Compare the digest recorded by Phase B on the RPi2 with the value recomputed on the Owner PC. They
**must** be identical — this is the direct regression check for the v0.3.17 failure. Record both
values in the ledger.

## 6. RPi2 offline install + runtime validation

Against the transferred wheelhouse only, with no network:

    --no-index --only-binary=:all: --require-hashes --find-links <wheelhouse>

Validate: install into an isolated venv; `pip check`; installed versions match
`requirements-armv7.lock` exactly; import every runtime package including native modules (CFFI,
cryptography, MarkupSafe, psutil, pydantic-core, uvloop, PyYAML); service starts; runtime smoke
checks pass; **rollback** restores the previous state; temporary venv removed. Docker stays stopped.

## 7. Immutable tag

Re-confirm `HEAD == origin/main == <CANDIDATE>`, clean tree, `APP_VERSION == 0.3.18`, and that
`v0.3.18` does not yet exist. Re-run the Owner exact-commit local preflight against `<CANDIDATE>`.
Only then create the **immutable annotated tag** `v0.3.18` on `<CANDIDATE>` and confirm it peels to
`<CANDIDATE>`. (Candidate-SHA CI already passed in step 1; the tag-triggered CI run afterwards is a
post-tag backstop only and cannot prevent a bad tag.)

## 8. SRT from the exact tag

Run the producer from the tag, passing the **required** transfer manifest:

    python release/ccc_release.py --version 0.3.18 --sign-key <key> \
        --git-ref v0.3.18 --wheelhouse-armv7 <bundle>/wheelhouse-armhf \
        --provenance-armv7 <bundle>/wheelhouse-armv7.json \
        --armv7-runtime-lock <bundle>/requirements-armv7.lock \
        --image-manifest <phase-a>/image-manifest.json \
        --transfer-manifest <manifest> \
        --recommended-core <core> --out dist

`--transfer-manifest` is mandatory: a missing, misnamed, substituted, malformed or mismatched
manifest fails closed **before** any artifact bytes exist. Signing happens only on the Owner PC.

## 10. RPi2 device qualification (armv7l)

Install/update from the **signed** armv7l artifact over SSH: signature and digest verification,
install, service start, runtime smoke checks, and a deliberate **rollback/recovery** exercise.

## 11. RPi4 device qualification (aarch64) — separate, not implied by RPi2

The aarch64 path is structurally different and must be qualified independently:

- the aarch64 artifact carries **no wheelhouse** (the producer rejects one);
- dependencies install **online** under the committed `requirements-aarch64.lock` —
  **network is a prerequisite**, unlike the RPi2 offline path;
- the Logical Tree Digest does **not** cover this path.

Validate: signature/digest verification of the exact signed aarch64 artifact; locked online
dependency installation; service start; runtime smoke checks; rollback/recovery on failed install.
**RPi2 success is not evidence for RPi4.**

## 12. Publish last

Only after every gate above is green: create the GitHub Release and upload exactly the four assets.
Verify each published digest equals the recorded digest and that the downloaded manifest signature
verifies. Then update ONLY the Status line of the v0.3.17 incident record (which remains *remediation in
progress* until this point) to `closed — superseded by v0.3.18`. Its facts, hashes and chronology
stay immutable.
